# Code Review

Fais une review complète du code
dans les fichiers modifiés.

Vérifie :
- La lisibilité
- Les erreurs potentielles
- Les bonnes pratiques
- La performance
▶ Claude Code
1 Créez .claude/commands/ ou .claude/skills/
2 Ajoutez un frontmatter YAML
3 Utilisez $ARGUMENTS pour les paramètres
4 Utilisez !`cmd` pour injecter du contexte
.claude/commands/fix-issue.md 
---
description: Corrige un issue GitHub
argument-hint: [issue-number]
allowed-tools: Read, Edit, Bash
model: sonnet
---

Corrige l'issue GitHub $ARGUMENTS.
Contexte : !`gh issue view $0`

# Invocation : /fix-issue 123
💡 Claude Code offre plus de contrôle (frontmatter, arguments, injection dynamique). Cursor est plus simple (Markdown pur). Les deux se versionnent dans Git.

Bundled Skills

Skills intégrées — Claude Code

🚀
/batch
Orchestrer des changements à grande échelle en parallèle. Décompose en 5-30 unités, chaque agent dans un git worktree isolé — implémente, teste et ouvre une PR.
/batch migrate src/ from Solid to React
✨
/simplify
Nettoyer le code récemment modifié. Lance 3 agents en parallèle (réutilisation, qualité, efficacité) — agrège et applique les corrections.
/simplify focus on memory efficiency
🔍
/debug
Diagnostiquer la session courante en lisant les logs de debug.
🔁
/loop
Exécuter un prompt à intervalle régulier — polling de déploiements.
/loop 5m check if deploy finished
📚
/claude-api
Charger la référence API Claude pour votre langage.
💡 Ces skills sont des playbooks — pas du code figé. Elles orchestrent des agents, lisent des fichiers et s'adaptent à votre codebase.

Comment créer un mode

1 Ouvrez Settings → Features → Chat
2 Cliquez Custom modes
3 Cliquez Add custom mode
4 Choisissez : nom, icône, raccourci clavier
5 Sélectionnez les outils activés
6 Ajoutez des instructions custom
🔮 À venir : .cursor/modes.json pour partager les modes en équipe via Git
