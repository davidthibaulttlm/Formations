Le Problème

Les limites des Rules

🎲

Probabiliste

Les Rules sont des suggestions — l'IA peut les ignorer ou les oublier en cours de session

🔓

Pas de garantie

Impossible de bloquer une commande dangereuse ou de forcer un formatage avec une simple Rule

📊

Pas d'audit

Aucun moyen de logger les actions de l'agent ou d'envoyer de la télémétrie

→ Hooks = du code qui s'exécute systématiquement, à chaque fois, de façon déterministe
Concept

Qu'est-ce qu'un Hook ?

Agent veut agir → Hook intercepte → ✅ Autoriser  |  ❌ Bloquer  |  📝 Modifier

Un hook est un processus externe (script, prompt LLM, endpoint HTTP) que l'outil exécute automatiquement à un moment précis du cycle de l'agent. Contrairement aux Rules qui sont des suggestions, les hooks sont garantis à 100%.

📏 Rules
Instructions que l'IA devrait suivre
Probabiliste — peut être ignoré
🪝 Hooks
Code qui s'exécute systématiquement
Déterministe — garanti à 100%
💡 Pensez aux hooks comme du middleware Express — ils interceptent les actions de l'IA au lieu des requêtes HTTP.

Configuration

Où configurer les hooks

⚡ Cursor
1 Créez .cursor/hooks.json (projet) ou ~/.cursor/hooks.json (global)
2 Listez les hooks par événement
3 Pointez vers un script exécutable
4 Cursor recharge automatiquement
{
  "version": 1,
  "hooks": {
    "afterFileEdit": [
      {
        "command": ".cursor/hooks/format.sh"
      }
    ]
  }
}
▶ Claude Code
1 Éditez .claude/settings.json (projet) ou ~/.claude/settings.json (global)
2 Ajoutez un matcher regex pour filtrer
3 Définissez le type : command, prompt, http, agent
4 Ou utilisez /hooks pour le menu interactif
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": ".claude/hooks/format.sh"
          }
        ]
      }
    ]
  }
}
💡 Différence clé : Claude Code utilise un matcher regex pour filtrer sur le nom de l'outil. Cursor applique le hook à TOUTES les occurrences de l'événement (sauf si matcher est défini).
Types

Les 4 types de hooks

⚙️
Command
Cursor + Claude Code
Script shell recevant du JSON sur stdin. Retourne exit 0 (OK) ou exit 2 (bloquer).
80% des cas
💬
Prompt
Cursor + Claude Code
Prompt envoyé à un LLM rapide pour une évaluation oui/non. Pas besoin de code.
Facile
🌐
HTTP
Claude Code
POST avec le JSON de l'événement vers une URL. Idéal pour services externes et compliance.
Intégration
🤖
Agent
Claude Code
Sous-agent multi-tour avec accès aux outils (Read, Grep). Pour vérifications complexes.
Puissant
💡 Commencez toujours par Command. N'utilisez Prompt/HTTP/Agent que quand le script ne suffit pas.
Exemple

Auto-format avec Prettier

.claude/settings.json Claude Code 
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "jq -r '.tool_input.file_path' | xargs npx prettier --write 2>/dev/null; exit 0"
          }
        ]
      }
    ]
  }
}
.cursor/hooks.json Cursor 
{
  "version": 1,
  "hooks": {
    "afterFileEdit": [
      {
        "command": ".cursor/hooks/format.sh"
      }
    ]
  }
}

💡 exit 0 garantit que le hook ne bloque jamais l'agent même si Prettier échoue. C'est le pattern fail-open.
Exemple

Bloquer les commandes dangereuses

.claude/hooks/block-rm.sh Bash 
#!/bin/bash
COMMAND=$(jq -r '.tool_input.command')

if echo "$COMMAND" | grep -q 'rm -rf'; then
  jq -n '{
    hookSpecificOutput: {
      hookEventName: "PreToolUse",
      permissionDecision: "deny",
      permissionDecisionReason: "Commande destructrice bloquée"
    }
  }'
else
  exit 0
fi
Script (Command)
Le plus précis — regex, parsing JSON, logique conditionnelle
exit 2 = bloquer
Prompt (LLM)
"Does this command look safe? Block rm -rf and DROP TABLE."
Pas de code
Cursor shortcut
"matcher": "rm -rf" dans beforeShellExecution — regex intégrée
Le plus simple
⚠️ L'agent reçoit la raison du blocage et peut s'adapter — il ne reste pas bloqué.

Bonnes Pratiques

Les 6 règles d'or

1

Fail-open par défaut

Terminez par exit 0 pour les hooks non-critiques. Réservez exit 2 pour les blocages intentionnels.

2

Rapides et légers

Un hook lent ralentit l'agent. Gardez sous 5 secondes. Utilisez timeout pour limiter.

3

Hooks projet dans Git

Partagez les hooks de sécurité via .claude/settings.json ou .cursor/hooks.json

4

Commencer par Command

Couvre 80% des besoins. Prompt/Agent uniquement quand le script ne suffit pas.

5

Tester avec des logs

Redirigez vers un fichier de log (>> /tmp/hooks.log) pendant le développement.

6

Matcher précis

Filtrez avec Write|Edit plutôt que .* pour éviter de ralentir l'agent.
