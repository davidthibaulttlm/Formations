Création

Comment créer des Rules

⚡ Cursor
1 Ctrl+Shift+P → New Cursor Rule
2 Nommez la rule (ex: react-patterns)
3 Choisissez le type d'application
4 Rédigez en Markdown
.cursor/rules/react-patterns.mdc 
---
description: Conventions React
globs: "**/*.tsx"
alwaysApply: true
---
# Conventions React
- Composants fonctionnels
- Named exports uniquement
▶ Claude Code
1 Créez CLAUDE.md à la racine du projet
2 Créez .claude/rules/
3 Ajoutez des fichiers .md avec frontmatter
4 Utilisez /memory pour la mémoire auto
.claude/rules/react-patterns.md 
---
paths:
  - "**/*.tsx"
---
# Conventions React
- Composants fonctionnels
- Named exports uniquement
💡 Le contenu des rules est identique — seul le format du frontmatter change : globs (Cursor) vs paths (Claude Code)

Migration

Migration depuis un CLAUDE.md monolithique

❌ Avant

📄 CLAUDE.md ~500 lignes
├── Style de code
├── Tests
├── API
├── Architecture
├── Git
└── ...tout mélangé 😵
⚠️ >300 lignes = l'IA ignore des instructions

✅ Après

📄 CLAUDE.md ~50 lignes
Stack + Commandes + Fichiers clés
📁 .cursor/rules/ Cursor
├── code-style.mdc
├── testing.mdc
└── api-patterns.mdc
📁 .claude/rules/ Claude Code
├── code-style.md
├── testing.md
└── api-patterns.md
1 Identifier les catégories dans votre CLAUDE.md
2 Extraire dans .cursor/rules/ et/ou .claude/rules/
3 Garder uniquement l'essentiel dans CLAUDE.md

Bonnes Pratiques

Les 7 règles d'or

1

Versionner dans Git

L'équipe en profite automatiquement

2

Commencer petit

Ajouter une rule quand l'agent fait 2× la même erreur

3

Être spécifique

"Named exports" plutôt que "Clean code"

4

Inclure le pourquoi

L'IA suit mieux quand elle comprend la raison

5

Max 500 lignes

Par fichier de rule — diviser si plus

6

Penser multi-outils

CLAUDE.md comme base + rules spécifiques

7

Tester dans les deux outils

Vérifier que l'agent respecte les rules dans Cursor ET Claude Code
